# Source Lab Pipeline Contract v0.3.0

> **Scope**: contract 문서만. 구현/DB migration/worker/router/UI/notification 없음.

---

## 1. Pipeline States

| State | 설명 | 전이 |
|-------|------|------|
| `url_intake` | URL 입력 접수. 중복/형식 검증 | → `low_level_analysis` |
| `low_level_analysis` | 메타데이터 추출, 콘텐츠 파싱, 기본 분석 | → `self_close` 또는 `judgment_requested` |
| `self_close` | low-model이 자체 판단 가능 → 자동 종료 | terminal |
| `judgment_requested` | 용용봇 판단 요청 큐에 등록 | → `judgment_result_received` |
| `judgment_result_received` | 용용봇 판단 결과 수신 | → `final_close` 또는 `queued_followup` |
| `final_close` | 최종 종료 | terminal |
| `queued_followup` | 후속 처리 큐 (용용봇이 추가 작업 지정) | terminal |

---

## 2. Branching Rule

### 2.1 Self-Close 조건 (low-model이 자체 판단 가능)

아래 **모두** 해당 시 `self_close`:

- 콘텐츠가 명확한 사실 기반 정보 (뉴스, 공지, 기술 문서 등)
- 감정/판단/의견이 핵심이 아닌 경우
- 신뢰도(confidence) ≥ 0.8
- 아래 "용용봇 판단 필요 영역"에 해당하지 않음

### 2.2 용용봇 Judgment Request 조건

아래 **하나라도** 해당 시 `judgment_requested`:

- **투자/재무 판단**: 주가 예측, 투자 의견, 밸류에이션 평가
- **정책/규제 해석**: 법률 자문, 규제 compliance 판단
- **개인 데이터 해석**: 개인정보 관련 민감 콘텐츠
- **모호한 사실 관계**: 상반된 정보, 팩트체크 필요
- **신뢰도 부족**: confidence < 0.8
- **용용봇 직접 지시**: 용용봇이 특정 패턴에 대해 직접 판단 요청 설정

### 2.3 Low-Model이 판단하면 안 되는 영역

| 영域 | 이유 |
|------|------|
| 투자 결정 | 재무적 리스크, 규제 책임 |
| 법률 해석 | 전문성 부족, 오류 시 피해 |
| 개인정보 처리 | 프라이버시 리스크 |
| 정치적 의견 | 편향 리스크 |
| 의료/건강 조언 | 전문성 부족, 안전 리스크 |

## 3. Payload Contract

### 3.1 Judgment Request Payload

```json
{
  "request_id": "uuid",
  "source_url": "string (required)",
  "content_summary": "string (required, 500자 이내)",
  "analysis_snapshot": {
    "title": "string",
    "content_type": "news | opinion | technical | financial | other",
    "key_claims": ["string"],
    "extracted_entities": ["string"],
    "raw_text_preview": "string (2000자 이내)"
  },
  "branch_reason": "enum (required)",
  "confidence": "float 0.0~1.0 (required)",
  "priority": "enum (required)",
  "requested_at": "ISO 8601 (required)",
  "requested_by": "string (agent_id, required)"
}
```

#### branch_reason enum

| 값 | 설명 |
|----|------|
| `investment_judgment` | 투자/재무 판단 필요 |
| `regulatory_interpretation` | 규제/법률 해석 필요 |
| `personal_data_sensitivity` | 개인정보 민감 콘텐츠 |
| `conflicting_information` | 상반된 정보 팩트체크 |
| `low_confidence` | 신뢰도 부족 |
| `explicit_yongyongbot_request` | 용용봇 직접 지시 패턴 |

#### priority enum

| 값 | 기준 |
|----|------|
| `urgent` | 실시간성이 중요한 이슈 (속보, 시장 급변동) |
| `high` | 투자 판단, 규제 해석 |
| `medium` | 팩트체크, 일반 판단 요청 |
| `low` | 후속 분석, 비긴급 |

### 3.2 Judgment Result Payload

```json
{
  "request_id": "uuid (required, request와 매칭)",
  "judgment": "enum (required)",
  "reason": "string (required, 판단 근거)",
  "confidence": "float 0.0~1.0 (required)",
  "evidence": ["string (참조한 근거 목록)"],
  "action": "enum (required)",
  "followup_tasks": [
    {
      "task_type": "string",
      "description": "string",
      "assignee": "agent_id"
    }
  ],
  "decided_at": "ISO 8601 (required)",
  "decided_by": "string (용용봇 agent_id, required)"
}
```

#### judgment enum

| 값 | 설명 |
|----|------|
| `approved` | low-model의 분석 결과 승인, 그대로 진행 |
| `rejected` | low-model의 분석 결과 거부, 재분석 필요 |
| `modified` | 수정된 판단 제공 |
| `escalated` | 사람(용진)에게 직접 확인 필요 |
| `deferred` | 추가 정보 수집 후 재판단 |

#### action enum

| 값 | 설명 |
|----|------|
| `close` | 최종 종료 |
| `reanalyze` | low-model 재분석 지시 |
| `queue_followup` | 후속 작업 큐에 등록 |
| `escalate_to_human` | 사람에게 에스컬레이션 |
| `archive` | 보관만 (추가 조치 없음) |

## 4. 용용봇 Queue 기준

### 4.1 용용봇이 반드시 봐야 하는 항목

| 항목 | 이유 |
|------|------|
| 모든 `judgment_requested` 항목 | 파이프라인 contract 상 용용봇 판단 필요 |
| priority `urgent` | 실시간 대응 필요 |
| priority `high` | 투자/규제 관련 고위험 |
| 용용봇이 직접 지시한 패턴 (`explicit_yongyongbot_request`) | 명시적 관심사 |

### 4.2 용용봇이 안 봐도 되는 항목

| 항목 | 이유 |
|------|------|
| `self_close` 처리된 항목 | low-model 자체 판단 완료 |
| confidence ≥ 0.8 + 민감 영역 미해당 | 자체 판단 충분 |
| 중복 URL (이미 분석 완료) | 재분석 불필요 |

### 4.3 우선순위 기준

```
urgent > high > medium > low
```

같은 priority 내에서는 `requested_at` 오래된 순 (FIFO).

### 4.4 사람이 읽을 최소 Summary 형식

용용봇이 큐에서 항목을 볼 때 최소한 다음 정보를 포함:

```
[URL] source_url
[유형] content_type
[요약] content_summary (3줄 이내)
[핵심 claims] key_claims (최대 5개)
[분기 사유] branch_reason
[신뢰도] confidence
[시간] requested_at
```

## 5. Non-Goals

아래 항목은 v0.3.0 범위 **밖**:

- ❌ 구현 코드 (worker, router, handler)
- ❌ DB migration / 스키마 적용
- ❌ Queue 생성 / 메시지 큐 설정
- ❌ Slack/notification wiring
- ❌ SourceLab UI 변경
- ❌ low-model 프롬프트 엔지니어링
- ❌ 성능/비용 최적화
